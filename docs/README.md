# Documentation Index

This directory contains the main documentation for my homelab infrastructure portfolio.

The repo is organized so the core infrastructure story stays easy to follow while individual projects have room to breathe. The goal is not to make this look like a generic lab template. This is documentation for a real home environment that I use, maintain, and keep improving.

## Documentation Standard

Each document should stay grounded in what is actually running or intentionally planned.

Completed work should be described as completed only when it has been implemented and tested. Planned work should stay clearly marked as planned. This matters because the strength of this portfolio is not pretending the homelab is perfect. The strength is showing that I can build, operate, document, troubleshoot, and improve real infrastructure over time.
